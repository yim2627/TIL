* Git 
	+  Push : local repo -> remote repo
	+  Pull : remote repp -> local repo
	+  Merge, Merge conflict
	+  Branch : 각각의 branch가 같은 위치에 코드를 수정하여 병합한다면 충돌이 일어남
	+  Commit
	+  Add  
	+  Fetch 
	+ working directory : 유저가 사용하고 있는 컴퓨터에 위치한 작업 파일을 말한다. 실제로 유저는 이 곳에서 작업을 함
	+ staging area : commit할 정보를 저장하는 임시 저장소
	+ local repo : 말 그대로 local 저장소, push하여 remote 저장소로 가기 전에 마지막으로 유저의 컴퓨터에 저장되는 저장소
	+ remote repo : 인터넷이나, 네트워크 등에 위치하는 저장소, local repo를 push하여 저장하고 pull하여 내려받을 수 있음

# 문제점 / 고민한 점 / 느낀 점
git 실습을 하는 도중, 오류가 발생했다.   
해결 방법을 찾으면서, 2021년 8월 13일 부로 GitHub에서 Git 작업을 인증 할 때 더 이상 계정 암호를 허용하지 않으며, ```token-based authentication```이 필요한 것을 알게되었고, 오류를 해결했다.   
천리길도 한걸음부터 라는 말이 있듯이 조급함과 욕심을 버리는 것부터 시작인 것 같다.   
내가 이전에 무엇을 쓰고 배웠든 내가 원하는 목표를 이룬 사람들보다 훨씬 부족함을 알고, 꾸준히 열심히 해야된다.   
상대방과 서로 코드 리뷰를 하면서 이렇게도 할 수 있구나 란 생각이 들었고, 상대방에게 배울 점도, 나의 고칠 점도 많았다.   
또한 경험해보지못한 페어 프로그래밍을 하면서, 긴가민가했던 지식도 서로 보충해주고 상기시켜주며, 처음 본 사람과도 재밌고 정말 유익하게 대화 할 수 있다는 점도 알게되었다.
